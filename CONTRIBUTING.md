# Contributing to Cybertica

Thank you for your interest in contributing to Cybertica! We welcome help in making the Local Talent Arena even better.

## How Can I Contribute?

### Reporting Bugs
If you find a bug, please create a GitHub issue and include:
- A clear, descriptive title.
- Steps to reproduce the issue.
- Expected vs. actual behavior.
- Flutter version (`flutter doctor` output).

### Suggesting Enhancements
We are always open to new features or ideas. Please open an issue outlining:
- The goal/problem this feature solves.
- A detailed description of the proposed enhancement.
- Mockups or designs if applicable.

### Submitting Pull Requests (PRs)
1. Fork the repository and create your branch from `main`.
2. Ensure the project builds successfully and the code matches our style.
3. Run `flutter analyze` to verify that there are no analyzer issues.
4. Format your Dart code using `dart format .`.
5. Submit a pull request with a clear description of your changes.

## Code of Conduct
Please be respectful and constructive in all communication and interactions with the community.
